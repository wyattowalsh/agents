# Skills-First Extension Decision Tree

## Rule

Prefer Agent Skills. Use MCP only for live/dynamic external state. Use native plugins only as harness-specific projections from canonical repo assets.

## Decision Tree

1. If the capability is instructions, deterministic scripts, references, templates, static analysis, local CLI automation, eval/checklist, or docs workflow, implement it as an Agent Skill.
2. If the capability requires browser/runtime state, authenticated SaaS, current search/docs, database/cloud/vector state, telemetry streams, or interactive external systems, evaluate MCP.
3. If the capability only exists to package, expose, or adapt canonical repo skills/MCP/config into a specific harness, implement it as a native plugin or harness adapter.
4. Prefer `npx` or `uvx` ephemeral invocation for safe tooling that does not require vendored source.
5. Pin versions or record source commits before promotion.
6. Never install community-discovered assets by default.

## Promotion Gates

- Source URL and pinned revision are recorded.
- License is compatible with this repository.
- Maintainer and activity posture are reviewed.
- Security and supply-chain risk is reviewed.
- CLI or MCP conformance fixtures exist.
- Rollback plan exists.
- OpenSpec task mapping exists.
- Docs and AI-instruction sync plan exists.

## Quarantine Triggers

- Auth bridging or credential reuse.
- Proxying model accounts or user sessions.
- Credential sharing or token extraction.
- Offensive security capability.
- Broad filesystem or shell access without sandbox policy.
- Unpinned binary or opaque installer.
