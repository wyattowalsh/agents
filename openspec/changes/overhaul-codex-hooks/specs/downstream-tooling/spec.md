# Downstream Tooling Delta

## MODIFIED Requirements

### Requirement: Codex Hook Projection Uses Current Hook Schema

The repository SHALL project repo-managed Codex hooks using the current Codex lifecycle hook schema while preserving local hook groups that are not generated by this repository.

#### Scenario: Rendering Codex hook handlers

- **WHEN** `scripts/sync_agent_stack.py` renders hooks for the Codex harness
- **THEN** supported logical events SHALL include `SessionStart`, `UserPromptSubmit`, `PreToolUse`, `PermissionRequest`, `PostToolUse`, `PreCompact`, `PostCompact`, `SubagentStart`, `SubagentStop`, and `Stop`
- **AND** each command handler SHALL include `type: "command"`, the rendered absolute command, and a seconds-based `timeout`
- **AND** a handler SHALL include `statusMessage` when the registry provides one or can derive one from the hook description.

#### Scenario: Merging live Codex hooks

- **GIVEN** live `~/.codex/hooks.json` contains local hook groups and older repo-generated groups
- **WHEN** the home sync updates Codex hooks
- **THEN** non-generated local hook groups SHALL be preserved
- **AND** groups containing repo-managed `wagents-hook.py` commands SHALL be replaced by the current generated groups.

#### Scenario: Continuing from a stop verifier

- **WHEN** a Codex `Stop` hook needs one more verification pass
- **THEN** the hook SHALL return JSON with `decision: "block"` and a human-readable `reason`
- **AND** it SHALL skip continuation when `stop_hook_active` is already true.

#### Scenario: Returning post-edit quality context

- **WHEN** a Codex `PostToolUse` hook observes a touched source path
- **THEN** the hook SHALL return context-only verification guidance
- **AND** it SHALL run only lightweight non-mutating checks on directly touched paths.

#### Scenario: Gating unsupported final code-change claims

- **WHEN** a Codex `Stop` hook observes a final assistant message that claims code or repository work changed without validation evidence
- **THEN** the hook SHALL ask Codex to continue with `decision: "block"` and a human-readable verification prompt
- **AND** it SHALL not recurse when `stop_hook_active` is already true.
