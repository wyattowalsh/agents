# MCP Audit

## Objective

Convert the repo MCP collection from an ad hoc list into a curated live-systems layer with portable install commands, explicit risk metadata, replacement-by-skill decisions, and CI scanning.

## Read order

1. `00-current-mcp-inventory.md`
2. `01-replacement-candidates.md`
3. `02-index-gapfill-candidates.md`
4. `03-security-and-sandboxing.md`
5. `04-conformance-fixtures.md`

## Non-goals

- Do not promote MCP servers solely because they appear in an index.
- Do not keep MCP servers for static knowledge or deterministic local workflows when a skill is simpler.
- Do not treat absolute local paths as portable.
