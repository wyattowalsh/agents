# Harness Registry

## Objective

Record what the repo claims to support and what is actually validated.

## Support tiers

- `validated`: implemented, tested, and documented.
- `repo-present-validation-required`: files/folders exist in the repo, but end-to-end conformance remains required.
- `planned-research-backed`: authoritative docs exist; repo support is planned.
- `experimental`: preview or unstable surface.
- `unverified`: repo/community signal exists, but no stable authoritative contract was verified.
- `unsupported`: intentionally out of scope.

## Source of truth

`planning/manifests/harness-registry.yaml` is the source of truth. Generated docs and README support tables must not be manually edited independently.
