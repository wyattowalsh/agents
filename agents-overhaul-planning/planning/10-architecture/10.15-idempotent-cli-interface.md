# Idempotent CLI Interface

## Objective

Design `wagents` as the stable automation facade for repo maintainers, CI, and harness-specific setup flows.

## Command families

```text
wagents doctor [--json]
wagents inventory [--json] [--write planning/manifests/repo-inventory.json]
wagents catalog list|search|explain [skills|mcp|plugins|harnesses]
wagents skills validate|audit|sync|install|update|remove|package
wagents mcp audit|inspect|scan|classify|render|sync
wagents harness detect|status|render|sync|rollback
wagents docs check|generate|diff|linkcheck
wagents openspec doctor|status|new|validate|graph|archive
wagents taskgraph validate|critical-path|assign|render
```

## CLI guarantees

- Every mutating command has `--dry-run` or `--preview`.
- Every mutating command emits a transaction ID.
- Every command intended for CI has `--json`.
- Exit code `0` means pass/no-op, `1` means validation failure, `2` means environment/prerequisite error, `3` means unsafe/unapproved mutation, `4` means network/source unavailable.
- Commands must not require interactive input in CI mode.

## Output contract

```json
{
  "schema_version": 1,
  "command": "wagents ...",
  "status": "pass|fail|warn|noop",
  "transaction_id": null,
  "findings": [],
  "artifacts": [],
  "next_actions": []
}
```

## Acceptance criteria

- CLI contract fixtures exist under `tests/fixtures/cli/`.
- Docs include command examples for human and CI usage.
- Dangerous commands demonstrate rollback path.
