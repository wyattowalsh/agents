# Security Model

## Threat model

The control plane handles untrusted external packages, scripts, MCP servers, and configuration writes into sensitive AI tools. Treat all external artifacts as code execution unless proven otherwise.

## Controls

| Area | Control |
|---|---|
| Skills | audit scripts, references, allowed tools, network behavior, destructive commands |
| MCP | env-only secrets, transport review, auth scope review, tool allowlists |
| Plugins | maintainer/license/provenance review and sandbox notes |
| Config writes | backups, diffs, rollback, no inline secrets |
| Docs | generated warnings and support badges |
| CI | secret scan, schema validation, clean-room tests |
| Provenance | checksums, lockfile, source URL, last verified timestamp |

## External artifact states

- `unseen`: discovered, not reviewed
- `metadata_reviewed`: catalog metadata captured
- `source_reviewed`: repo/source skimmed
- `sandbox_tested`: installed in clean temp environment
- `supported`: passed conformance and docs gates
