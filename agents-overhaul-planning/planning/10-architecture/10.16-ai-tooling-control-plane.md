# AI Tooling Control Plane

## Objective

Unify skills, MCP servers, plugins, instructions, agent profiles, OpenSpec changes, and docs under one control plane.

## Layers

```text
L0 Source Ledgers      external docs, official specs, indexes, repo facts
L1 Registries          harness, skill, MCP, plugin, docs, task graph
L2 Policy              support tiers, install policy, security rules, provenance
L3 Projection          harness adapters, templates, golden fixtures
L4 Transaction Engine  preview, backup, apply, validate, rollback
L5 UX Surface          CLI, optional dashboard, generated docs, OpenSpec UX
L6 Observability       audit logs, drift reports, run graphs, cost/latency traces
```

## Control-plane invariants

1. Registry IDs are stable and kebab-case.
2. Docs generated from registries must not be edited manually unless marked source-owned.
3. Skills-first decision tree is enforced in CI for new extension proposals.
4. MCP servers require dynamic-state justification.
5. Every support-tier upgrade requires evidence artifact.
6. Every external component has provenance and risk metadata.
