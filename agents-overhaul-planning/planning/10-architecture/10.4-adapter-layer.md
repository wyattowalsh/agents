# Adapter Layer

## Adapter responsibilities

Each adapter should expose:

```python
class HarnessAdapter(Protocol):
    id: str
    def detect(self) -> DetectionResult: ...
    def render(self, registry: Registry) -> list[Projection]: ...
    def diff(self, projection: Projection) -> ConfigDiff: ...
    def apply(self, transaction: ConfigTransaction) -> ApplyResult: ...
    def validate(self) -> ValidationResult: ...
    def rollback(self, snapshot_id: str) -> RollbackResult: ...
```

## Adapter boundaries

Adapters should not decide whether a capability is first-class. They should consume canonical registries and render harness-specific files.

## Compatibility outputs

Each adapter emits:

- config snippets
- path manifest
- unsupported field warnings
- install commands
- conformance fixture
- docs summary

## OpenSpec integration

Adapters must read OpenSpec support status and avoid projecting planned-only capabilities as implemented.

## Merge strategy

- One adapter PR per harness.
- Core model changes merged first.
- Fixtures and docs can land in parallel after registry schema stabilizes.
