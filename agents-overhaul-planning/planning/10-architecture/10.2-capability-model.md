# Capability Model

## Canonical capability types

| Type | Use case | Examples | Preferred install |
|---|---|---|---|
| `instruction` | Always/mostly-on behavioral guidance | `AGENTS.md`, `instructions/global.md` | generated files/symlinks |
| `openspec_workflow` | durable change intent and requirements | proposal/design/tasks/spec deltas | `openspec/changes/*` |
| `skill` | portable task-specific know-how | `skills/<name>/SKILL.md` | `npx skills add` or repo-local copy |
| `plugin` | harness-native bundled behavior | Claude/OpenCode/Codex plugins | harness plugin manager |
| `mcp` | live tool/data access | Context7, browser, GitHub, telemetry | `uvx`/`npx`/remote Streamable HTTP |
| `agent` | role/persona/subagent | docs-writer, code-reviewer | harness-specific profile |
| `hook` | lifecycle automation | pre/post tool hooks | harness-specific settings |
| `dashboard` | user-facing catalog/control UX | docs site, CLI browser | generated docs + CLI |
