# Architecture Principles

## Principles

1. **Spec-governed:** important changes start as OpenSpec changes.
2. **Skills-first:** use portable `SKILL.md` capabilities wherever possible.
3. **MCP-when-live-state:** MCP is for live external systems, not static knowledge.
4. **Generated, not copied:** docs, snippets, and support matrices come from registries.
5. **Transaction-safe:** never mutate user configs without backup and validation.
6. **Support-tier honest:** distinguish first-class, validated, experimental, and watchlist.
7. **Harness-specific where necessary:** do not erase real client differences.
8. **Small PRs:** design for many parallel PRs with isolated file ownership.
9. **Security before convenience:** external artifacts are untrusted until reviewed.
10. **Progressive disclosure:** minimize always-loaded context.

## Non-goals

- Building a full multi-agent runtime.
- Replacing harness-native functionality.
- Turning all external tools into MCP servers.
- Treating public marketplaces as trusted authorities.
