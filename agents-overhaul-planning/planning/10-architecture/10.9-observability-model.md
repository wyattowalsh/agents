# Observability Model

## Objective

Make installs, syncs, validation runs, skill audits, and MCP checks inspectable without leaking secrets.

## Events

Emit structured events for:

- harness detection
- registry loading
- config rendering
- dry-run diffs
- apply transactions
- rollback transactions
- skill validation
- MCP smoke tests
- OpenSpec validation
- docs generation

## Event schema

```json
{
  "event": "wagents.config.apply",
  "timestamp": "ISO-8601",
  "harness": "claude-code",
  "transaction_id": "...",
  "status": "success|failure|rollback",
  "changed_paths": [],
  "redacted": true
}
```

## Metrics

- install success rate by harness
- config drift count
- supported/experimental/watchlist counts
- MCP smoke-test failure rate
- stale docs detection rate
- skill validation failures
- rollback rate

## Privacy

Do not log:

- secret values
- OAuth tokens
- raw user prompts unless explicitly enabled
- full file contents unless explicitly part of a test fixture
