# Canonical Manifest Contracts

## Objective

Create machine-readable manifests that can drive installs, docs, CI conformance, audits, and support matrices.

## Manifest set

| Manifest | Purpose | Generated? | Blocking validation |
|---|---|---:|---:|
| `repo-inventory.json` | Latest repo tree, generated/source classification | Yes | Yes |
| `harness-registry.yaml` | Harness support tiers and projection surfaces | Mixed | Yes |
| `skill-registry.yaml` | Local and external skill metadata | Mixed | Yes |
| `mcp-registry.yaml` | Local and candidate MCP metadata | Mixed | Yes |
| `plugin-registry.yaml` | Native plugin bundles and marketplaces | Mixed | Yes |
| `external-tool-catalog.json` | Research-sourced ecosystem ingredients | Yes | No |
| `docs-artifacts-manifest.yaml` | Docs source/generator mapping | Mixed | Yes |
| `source-ledger.json` | External source URLs and freshness | Yes | No |
| `task-graph.json` | Subagent tasks and dependencies | Yes | Yes |

## Required common fields

```yaml
id: string-kebab-case
name: human-readable name
kind: skill|mcp|plugin|harness|doc|policy|adapter|tool
support_tier: validated|repo-present-validation-required|planned-research-backed|experimental|unverified|unsupported
source_refs: []
repo_paths: []
owner_team: string
last_verified: YYYY-MM-DD|null
stability: stable|preview|beta|experimental|unknown
risk: low|medium|high|critical
validation_gates: []
```

## Skill-specific fields

```yaml
skill:
  directory: skills/<name>
  spec_version: agentskills.io
  has_scripts: bool
  has_references: bool
  has_assets: bool
  cli_contract:
    deterministic: bool
    supports_json_output: bool
    exit_codes_documented: bool
    idempotent: bool
  provenance:
    source: local|github|skills.sh|gh-skill|manual
    repository: string|null
    ref: string|null
    tree_sha: string|null
    checksum: string|null
```

## MCP-specific fields

```yaml
mcp:
  command: string|null
  args: []
  transport: stdio|streamable-http|sse|unknown
  auth_model: none|env|oauth|api-key|helper|unknown
  dynamic_state_required: bool
  replacement_assessment: keep-mcp|replace-with-skill|hybrid-skill-wrapper|dedupe-review|reject
  sandbox_profile: none|read-only|local-network|networked|browser|sensitive
```

## Acceptance criteria

- Manifests validate with JSON Schema/Pydantic models.
- Human docs are generated from manifests where feasible.
- Manifest IDs match task-graph IDs or explicit references.
- Each `validated` item has at least one CI gate and artifact.
