# Skill Packaging Model

## Goal

Make Agent Skills the default cross-harness extension primitive.

## Required structure

```text
skills/<skill-name>/
  SKILL.md
  references/
  scripts/
  assets/
```

## Skill authoring rules

- `name` must match directory name.
- `description` must explain what the skill does and when to use it.
- Keep `SKILL.md` concise and move detail to `references/`.
- Prefer deterministic scripts with JSON output and explicit exit codes.
- Include `metadata.version`, `metadata.source`, `metadata.trust_tier`, and `metadata.support_tier` in internal packaging extensions.

## CLI robustness contract

Scripts included in skills should support:

- `--help`
- `--json` where meaningful
- no secret printing
- idempotent dry-runs where possible
- non-zero exit codes on failure
- predictable stdout/stderr separation

## Install preference

1. `npx skills add github:wyattowalsh/agents --all ...`
2. project-local `.agents/skills/<name>/SKILL.md`
3. harness-specific compatibility path only if required
