# External Ecosystem Utilization Architecture

## Objective

Make the agents repo systematically benefit from the broader agent ecosystem without turning community discoveries into unscreened runtime dependencies.

## Principles

1. **Skills first**: if a capability can be represented as instructions + deterministic scripts + references/assets, package it as an Agent Skill.
2. **MCP second**: use MCP only for live state, authenticated external systems, browser/runtime control, current web/search/docs, databases/cloud resources, or telemetry streams.
3. **Plugins as projections**: harness-native plugins should project canonical registry state, not fork behavior.
4. **Community indexes are discovery inputs**: Glama, MCP.so, PulseMCP, and awesome lists are not trust roots.
5. **Promotion requires evidence**: license, security, maintenance, conformance, rollback, docs, and OpenSpec mapping.

## Integration Tiers

| Tier | Use |
|---|---|
| `catalog-only` | Listed in manifests for discovery; never installed. |
| `audited-reference` | Extracted design/architecture/UX patterns only. |
| `skill-candidate` | Candidate for `skills/external-*` or curated skill bundle. |
| `skill-wrapper` | A local skill wraps a third-party CLI/MCP/tool. |
| `mcp-profile` | Guarded MCP profile with explicit secrets, sandbox, and smoke fixtures. |
| `native-plugin-projection` | Harness plugin generated from canonical manifests. |
| `quarantine` | Security-sensitive; accessible only to red-team or explicit opt-in flows. |

## Architecture Pattern

```text
external discovery
  -> external-repo-evaluation manifest
  -> intake subagent review
  -> license/security/provenance review
  -> skill/MCP/plugin decision tree
  -> OpenSpec child change
  -> conformance fixture
  -> docs/instruction sync
  -> optional install/catalog promotion
```

## Reusable Patterns to Extract

- Multi-harness installer matrices from skill packs such as `caveman`, `graphify`, and `open-design`.
- Context graph generation and query flows from `graphify`, `codebase-memory-mcp`, `code-review-graph`, and `soulforge`.
- Session replay and run graph patterns from `claude-replay`, `codeg`, and `agentlytics`.
- Team dashboard/kanban patterns from `claude_agent_teams_ui`, `open-cowork`, and `agtx`.
- Skill registry validation models from `tech-leads-club/agent-skills`, `microsoft/skills`, `supabase/agent-skills`, `google/skills`, and `cloudflare/skills`.
- Package manager UX patterns from `OpenPackage`, `skillkit`, and Claude plugin/skill registries.
