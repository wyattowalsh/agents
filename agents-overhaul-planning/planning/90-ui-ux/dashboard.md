# Dashboard

## Objective

Improve dashboard for the control plane.

## Required flows

- detect current state
- explain desired state
- preview changes
- apply safely
- validate
- rollback if needed

## Acceptance criteria

- Flow is available through CLI or generated docs.
- Errors include actionable remediation.
- Support tier and trust tier are visible.
- Experimental capabilities require explicit opt-in.
