# Curated Awesome Lists and Community Aggregations

## Objective

Use curated lists to discover tools, not to establish support. Every item still requires primary-source validation.

## Target list categories

| Category | Extracted ingredients |
|---|---|
| Awesome AI Agents | frameworks, orchestration patterns, task decomposition models |
| Awesome MCP | MCP server categories, transport patterns, gaps in current collection |
| Awesome LLM / LLMOps | evals, observability, prompt management, tracing |
| Awesome AI IDE / DevTools AI | UI and editor extension patterns |
| Awesome Agent Frameworks | tool abstraction and agent graph patterns |
| Awesome LLM Security | prompt injection, sandboxing, policy, data leakage controls |
| Awesome Policy-as-Code | OPA/Rego, Cedar, Conftest-style checks for config governance |
| Awesome OpenTelemetry | tracing conventions and exporters |
| Awesome Prompt Engineering | prompt/version-control and reusable instruction packs |
| Awesome Copilot | Copilot skills, agents, chat modes, instructions, hooks, and community conventions |

## Screening rubric

- maintained in the last 90-180 days
- license is clear
- active maintainers or vendor ownership
- production usage or strong community signal
- documented install path
- documented security model
- can be integrated as skill/plugin/MCP without bespoke brittle glue

## Outputs

- `external-tool-shortlist.md` for promoted candidates
- registry watchlist rows
- OpenSpec proposal only for candidates intended for real support
