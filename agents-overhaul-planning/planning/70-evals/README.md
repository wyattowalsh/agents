# Evals Plan

## Objective

Create repeatable evaluations for skills, harness projections, docs generation, and orchestration quality.

## Eval categories

- skill activation correctness
- skill output quality
- harness config renderer correctness
- OpenSpec task decomposition quality
- MCP selection discipline
- docs completeness
- security refusal/containment behavior

## Integration

Evals should run locally in small smoke mode and in CI for critical skills/adapters.
