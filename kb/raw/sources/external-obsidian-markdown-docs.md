## Summary

Obsidian Help, CommonMark, YAML, and Git docs provide the evergreen foundation for this KB's vault conventions. Obsidian Help is primary for local Markdown vault storage, `.obsidian` settings, properties, aliases, wikilinks, and embeds. CommonMark is the portability floor for Markdown parsing; YAML 1.2.2 is the formal data-language source behind frontmatter syntax; Git `gitignore` is primary for ignore semantics. Dataview docs are relevant only as community-plugin primary material for queryable metadata compatibility.
