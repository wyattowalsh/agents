# Visual Inspiration

Use inspiration to extract direction, not to copy trade dress.

## Safe Inspiration Workflow

1. Identify why the reference is relevant: audience, surface, density, motion,
   typography, spatial system, visual metaphor, or interaction pattern.
2. Extract abstract traits: "narrow editorial measure", "single accent color",
   "grid-visible hierarchy", "large product stills", "quiet table controls".
3. Translate traits into project-local tokens, components, copy, and layout.
4. Change enough constraints that the result belongs to the current product.
5. Verify the design does not copy logos, proprietary illustrations, screenshots,
   signature layouts, brand colors, or distinctive component shapes.

## Extraction Table

| Reference Trait | Abstract Rule | Local Translation |
| --- | --- | --- |
| Oversized product photo | Product is the hero evidence | Use current product screenshot or generated local mock, not stock |
| Swiss grid | Alignment carries hierarchy | Use 12-column grid, narrow measure, one accent |
| Strong editorial type | Reading rhythm matters | Pair display/body roles and keep line length controlled |
| High-polish SaaS dashboard | Scan speed matters | Use quiet surfaces, compact controls, tabular numbers |
| Motion showcase | Motion teaches state | Animate origin/state change, not decorative loops |

## No-Copy Workflow

1. Name the source only as an influence, not as the target to clone.
2. Extract 3-5 abstract traits: pacing, density, hierarchy, material, motion,
   proof, or content strategy.
3. Translate each trait through the project domain, audience, and existing
   design system.
4. Deliberately change brand-owned specifics: colors, marks, typography,
   component silhouettes, composition, illustration style, screenshot framing,
   and copy structure.
5. Verify the result can be described without naming the inspiration source.

## Trait Translation Examples

| Borrowed Trait | Safe Translation |
| --- | --- |
| Premium feel | Higher whitespace discipline, stronger type scale, fewer visible choices, better proof hierarchy. |
| Stripe-like clarity | Clear product proof, comparison tables, calm spacing, direct action hierarchy; not Stripe gradients, layout, or icon style. |
| Swiss editorial | Grid rigor, restrained accent, readable type rhythm, large factual imagery. |
| Playful launch | One signature interaction or asset tied to the product, not generic blobs or gradients. |

## Review Checklist

- The design thesis stands on its own without copying a named brand.
- Color, type, layout, motion, imagery, and copy are domain-specific.
- The implementation reuses local tokens/components unless the brief justifies a
  new direction.
- Any badge, logo, screenshot, or partner mark is real, allowed, and linked to
  the correct source.

## Guardrails

- Do not copy a site's color palette, type pairing, exact composition, or visual
  identity unless the user owns it and asks for it.
- Avoid stock-looking atmosphere when the user needs to inspect a real product.
- If a design cannot be described without naming the reference brand, it is too
  close.
- Use generated bitmap images or real product images for hero media when a site
  needs visual assets; do not substitute abstract SVG decoration.
