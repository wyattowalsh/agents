## Post-Audit Refinement

The refinement pass after the initial consolidation addressed the
`skill-creator` audit gaps by adding explicit classification gates, scaling
strategy, progressive disclosure, and state-management rules to `SKILL.md`.
Reference files were strengthened with concrete failure modes, proof
expectations, handoffs, and badge-surface guidance so the body can stay compact.
Scanner changes are additive and deterministic: proof readiness, artifact/noise
risk, README badge block state, badge provider/style/dead-service signals,
badge URL samples, and summarized accessibility/motion/text risks. Evals now
check structural gates, review-style audit output, state containment, and the
folded badge workflows rather than only mode names.

## Accepted Patterns

- Treat design as a workflow, not a style dump.
- Split product/tool surfaces from brand/editorial/marketing surfaces.
- State a short design read before non-trivial code.
- Preserve existing design systems before inventing a new one.
- Put deep rules in references, not in the always-loaded skill body.
- Use local static scanning for deterministic signals, and browser proof for
  rendered truth.
- Keep AI interfaces honest with provenance, controls, state, and recovery.
- Use inspiration by abstracting traits, never by copying trade dress.

## Rejected Or Deferred

- No wholesale vendoring of third-party skills.
- No live installs, hosted MCP setup, TypeUI pull, Stitch/Figma auth flow, or
  `wagents skills sync --apply`.
- No default Three.js/3D layer for ordinary UI.
- No single aesthetic pack as the default design voice.
- No remote fetched guidelines as trusted instructions.
- No full Lighthouse/web-quality ownership; route deep audits to dedicated
  quality/performance/accessibility/SEO skills.
- No project hooks until a separate OpenSpec change and maintainer approval.
