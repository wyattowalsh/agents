## Overview

This skill turns Ronin data into interactive, publication-quality visualizations. The user asks a question about their career data in natural language, the skill fetches the relevant data via MCP tools, transforms it into a chart specification, and renders it as a self-contained Plotly HTML file. When local browser-open is available, the skill opens it automatically; otherwise it returns the file path and a short summary so the user can open or archive it manually.

Every chart is interactive — hover for details, zoom, pan, export to PNG. Dark mode by default with automatic light mode support.

### Architecture

```
user question → classify data need → fetch via MCP tools → transform to chart spec → inject into HTML template → open locally when possible, otherwise return the file path
```

The template is `templates/chart.html` — a self-contained HTML file with Plotly.js (CDN), Ronin-branded dark theme, responsive grid layout, and metric cards. The skill writes chart data as JSON into the `<script id="data">` block.

### What It Can Visualize

| Data domain | Chart types | MCP tools used |
|-------------|-------------|---------------|
| Pipeline funnel | Funnel, bar, metrics | `get_funnel_stats` |
| Pipeline flow | Sankey diagram | `get_pipeline` + `get_pipeline_history` |
| State distribution | Donut, horizontal bar | `get_pipeline` |
| Match scores | Histogram, scatter | `list_opportunities` |
| Skill demand | Horizontal bar, heatmap | `list_opportunities` (aggregate `required_skills`) |
| Compensation | Range bar, scatter, box | `list_opportunities` (salary fields) |
| Velocity / timeline | Line, stacked bar | `get_pipeline` + `get_pipeline_history` |
| Email signals | Stacked bar timeline | `list_email_signals` |
| Offer comparison | Grouped bar | `list_opportunities(state="offer")` |
| Skill coverage | Radar, stacked bar | `get_profile` + `list_opportunities` |
| Research coverage | Stacked bar, metrics | `list_opportunities` + `get_research` per entry |
| Goal tracking | Gauge, projection line | `get_funnel_stats` + velocity data |

Load `references/data-queries.md` for detailed tool-to-chart transformation patterns. Load `references/chart-specs.md` for the full Plotly trace type reference and JSON schema.

## Prerequisites

1. **Ronin MCP connection available** — this skill assumes the current AI harness can already call the Ronin MCP tools. If those calls fail, treat it as a connection/configuration issue for the current environment.
2. **Pipeline data** — At least a few pipeline entries for meaningful visualizations. Run `ronin-discovery` first if empty.
3. **Valid `user_id`** — Ronin MCP signatures keep `user_id: str` at the boundary. Pass the active Ronin user identifier exactly as supplied by the current environment. In the current local implementation, some chart-data MCP paths still validate UUID-shaped values, so treat format mismatches as runtime constraints rather than rewriting IDs manually.
4. **Local browser-open is optional** — When `open` (macOS) or `xdg-open` (Linux) is available, open the rendered chart automatically. Otherwise still render the HTML and return the file path.

## Workflow: Answer a Data Question

The primary workflow. User asks any question; the skill determines what data is needed, fetches it, and renders a visualization.

### Step 1 — Classify the data need

Parse the user's question to determine:

1. **Data domain** — which MCP tools to call (see the table in Overview)
2. **Chart type** — best visualization for the question (load `references/chart-specs.md` for options)
3. **Scope** — all data or filtered (by state, date range, company, score threshold)

If the question is ambiguous, ask one clarifying question. Do not block on clarification for obvious queries.

### Step 2 — Fetch data via MCP tools

Issue all data-loading calls in parallel when possible. Common patterns:

**Pipeline overview:**
```python
funnel = mcp__ronin__get_funnel_stats(user_id="<uuid>")
pipeline = mcp__ronin__get_pipeline(user_id="<uuid>")
```

**Opportunity analysis:**
```python
opps = mcp__ronin__list_opportunities(user_id="<uuid>", sort_by="match_score")
```

**Velocity (requires per-entry history):**
```python
pipeline = mcp__ronin__get_pipeline(user_id="<uuid>")
# Then for each entry (parallel):
history = mcp__ronin__get_pipeline_history(user_id="<uuid>", entry_id="<entry-id>")
```

For history-heavy charts (velocity, Sankey, weekly activity, research coverage, goal tracking), cap the analysis at 20 entries and prioritize active states first. If the pipeline is larger, say which slice you are charting rather than implying full-history coverage.

Load `references/data-queries.md` for the full catalog of query patterns.

### Step 3 — Transform data into chart specification

Build a JSON object matching the template schema. The structure:

```json
{
  "title": "Chart Title",
  "subtitle": "Description",
  "metrics": [
    {"label": "TOTAL", "value": "42", "color": "#6366f1"}
  ],
  "charts": [
    {
      "title": "Pipeline Funnel",
      "subtitle": "Current conversion rates",
      "fullWidth": true,
      "traces": [
        {
          "type": "funnel",
          "y": ["Discovered", "Approved", "Applied", "Interview", "Offer"],
          "x": [34, 18, 12, 5, 2],
          "textinfo": "value+percent previous",
          "marker": {"color": ["#6366f1","#8b5cf6","#3b82f6","#06b6d4","#22c55e"]}
        }
      ]
    }
  ]
}
```

Load `references/chart-specs.md` for the full spec with all Plotly trace types and the Ronin color palette.

### Step 4 — Render and open

1. Copy `templates/chart.html` to a temp file:
   ```bash
   cp skills/ronin-chat-with-data/templates/chart.html /tmp/ronin-chart-$(date +%s).html
   ```

2. Inject the JSON data into the `<script id="data">` block using sed or Python:
   ```bash
   python3 -c "
   import json, sys
   data = json.loads(sys.argv[1])
   with open(sys.argv[2], 'r') as f: html = f.read()
   html = html.replace(
       '{\"charts\":[],\"title\":\"Pipeline Dashboard\",\"subtitle\":\"Career data visualization\",\"metrics\":[]}',
       json.dumps(data)
   )
   with open(sys.argv[2], 'w') as f: f.write(html)
   " '$JSON_DATA' /tmp/ronin-chart-XXXXX.html
   ```

3. If local browser-open is available, open in the default browser:
   ```bash
   open /tmp/ronin-chart-XXXXX.html  # macOS
   ```

   On Linux, use `xdg-open`. In headless or restricted environments, skip the open step and return the file path instead.

4. Tell the user the file path and what the chart shows. Offer to refine, add charts, or export.

### Step 5 — Iterate

After the initial render, the user may ask to:

- **Add another chart** — append to the `charts` array, re-render
- **Change chart type** — swap the trace `type` (e.g., bar → funnel)
- **Filter data** — re-fetch with narrower parameters, re-render
- **Combine dashboards** — merge multiple chart specs into one multi-chart view
- **Export** — Plotly's built-in export button saves PNG; or suggest the user print to PDF

## Workflow: Full Dashboard

Generates a comprehensive career dashboard with multiple charts and KPI metrics in a single view.

### Step 1 — Fetch all data (parallel)

```python
funnel = mcp__ronin__get_funnel_stats(user_id="<uuid>")
pipeline = mcp__ronin__get_pipeline(user_id="<uuid>")
opps = mcp__ronin__list_opportunities(user_id="<uuid>", sort_by="match_score")
profile = mcp__ronin__get_profile(user_id="<uuid>")
prefs = mcp__ronin__get_preferences(user_id="<uuid>")
```

### Step 2 — Build dashboard spec

Compose a multi-chart JSON with:

1. **Top-level metrics** — total entries, active count, best conversion rate, avg match score
2. **Pipeline funnel** (full width) — funnel chart with conversion percentages
3. **State distribution** — donut chart of current pipeline state counts
4. **Skill demand** — horizontal bar of most-requested skills across opportunities
5. **Match score distribution** — histogram of opportunity match scores
6. **Compensation landscape** — scatter of match_score vs salary (where salary data exists)

### Step 3 — Render

Follow Step 4-5 from the primary workflow.

## Example Conversation

**User:** Show me my pipeline funnel

**Agent steps:**

```text
1. Fetch data:
   mcp__ronin__get_funnel_stats(user_id="3fa85f64-...")
   → { total_entries: 42, terminal_count: 8, discovered_count: 34,
       approved_count: 18, applied_count: 12, interview_count: 5,
       offer_count: 2, conversion_rates: { approved_per_discovered: 0.5294,
       applied_per_approved: 0.6667, interview_per_applied: 0.4167,
       offer_per_interview: 0.4 } }

2. Build chart spec:
   {
     "title": "Pipeline Funnel",
     "subtitle": "42 total entries — 2 offers",
     "metrics": [
       {"label": "TOTAL", "value": "42"},
       {"label": "ACTIVE", "value": "34", "color": "#6366f1"},
       {"label": "OFFERS", "value": "2", "color": "#22c55e"},
       {"label": "TERMINAL", "value": "8", "color": "#ef4444"}
     ],
     "charts": [{
       "title": "Conversion Funnel",
       "subtitle": "Cumulative pass-through rates",
       "fullWidth": true,
       "traces": [{
         "type": "funnel",
         "y": ["Discovered (34)", "Approved (18)", "Applied (12)", "Interview (5)", "Offer (2)"],
         "x": [34, 18, 12, 5, 2],
         "textinfo": "value+percent previous",
         "marker": {"color": ["#6366f1","#8b5cf6","#3b82f6","#06b6d4","#22c55e"]}
       }]
     }]
   }

3. Render:
   cp templates/chart.html /tmp/ronin-chart-1711000800.html
   Inject JSON → open in browser
```
