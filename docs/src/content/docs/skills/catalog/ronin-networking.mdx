## What It Does

A skill for cross-referencing professional contacts with pipeline opportunities, generating personalized outreach messages, and identifying networking gaps across the Ronin job search pipeline. Uses the profile as the single source of truth for shared background, company research for personalization hooks, and user memory for relationship history. Extended thinking (opus + 1M budget) for outreach synthesis. Pairs with the current contacts page UI (`web/app/(app)/contacts/page.tsx`), which is still browser-local rather than a backend-native contacts system.

## Critical Rules

1. Never fabricate shared background — report "limited personalization available" when data is sparse
2. Apple Mail MCP is optional — never block a workflow on its availability
3. Match contacts to opportunities by company name using case-insensitive comparison
4. Flag near-matches for user confirmation (e.g. "Google" vs "Alphabet") — do not auto-match ambiguous names
5. Do not fabricate contacts — only work with contact data the user explicitly provides
6. Always load pipeline opportunities before cross-referencing — an empty pipeline produces no matches
7. Include a clear call-to-action in every outreach message

### Scope Boundaries

This skill is not for automated discovery, email monitoring, or direct pipeline state management beyond what outreach requires. Not when the MCP server is unreachable — verify prerequisites first.

Classify the ask before loading tools:

- **Contact list, "who can help?", "who do I know at X?"** — Run Contact-Opportunity Cross-Reference.

If the request is ambiguous and includes both contacts and pipeline context, start with cross-reference. After you surface matched contacts or gaps, ask whether the user wants outreach drafted for any specific match.

## Overview

Networking is cross-cutting — useful at any pipeline state from `discovered` through `offer`. Contact records include fields such as name, company, email, phone, LinkedIn URL, title, relationship type (recruiter / hiring_manager / referral / colleague / mentor / other), notes, and how-we-met context.

This skill bridges the gap between the contacts page and the pipeline by:

1. **Cross-referencing** contacts against pipeline opportunities by company name
2. **Generating** personalized outreach messages grounded in the user's profile and company research
3. **Identifying** networking gaps — pipeline companies where the user has no contacts
4. **Tracking** relationship context through user memory and contact notes

### Design Principles

- **Leverages existing UI** — contacts are managed in the contacts page; this skill reasons over contact details the user provides and does not create new frontend pages or DB models.
- **Apple Mail is optional** — outreach generation works without Apple Mail MCP. When available, it can send messages directly. When unavailable, it produces copy-ready message text.
- **Null over confabulation** — when profile data is sparse or company research is unavailable, the skill reports gaps explicitly rather than fabricating shared background or personalization hooks.
- **Flat MCP tool parameters** — all tool calls use the documented parameter signatures exactly. No nested objects in parameters.

### Scaling strategy

- `list_opportunities(user_id)` is capped at 50 results. If the pipeline is larger, say so explicitly and batch by urgency (`state`) or relevance (`min_score`) instead of implying full coverage.
- For large contact imports, normalize and deduplicate by email first, then by `(name, company)` when email is missing. Tell the user when duplicate contacts were collapsed.
- Batch contact cross-reference work when the user provides a long list; do not silently drop contacts.
- Near-matches such as parent/subsidiary or abbreviation cases stay provisional until the user confirms them.

## Prerequisites

### 1. Ronin MCP connection

This skill assumes the current AI harness can already call the Ronin MCP tools.
If those calls fail, treat it as an MCP connection/configuration issue for the
current environment rather than assuming a Docker-first local setup.

### 2. Pipeline with opportunities

At least one opportunity must exist in the pipeline. Contacts are matched by company name against `opportunity.company`. Run `ronin-discovery` first if the pipeline is empty.

### 3. Contact data

This skill cannot directly read browser state. The user provides contact details either:
- By describing them in conversation (name, company, email, etc.)
- By pasting/exporting contact details from the contacts page UI

### 4. Apple Mail MCP (OPTIONAL -- not required)

Apple Mail MCP is **not required** for any workflow. All outreach generation works without it. When available, the skill can send messages directly and search prior correspondence. When unavailable, it produces copy-ready message text.

To check availability, attempt a probe call early in the session:

```python
mcp__apple_mail__search_emails(query="test")
```

A connection error or tool-not-found means Apple Mail MCP is unavailable — proceed without it. Do not retry or block any workflow on this.

## Workflow: Contact-Opportunity Cross-Reference

Match the user's contacts against pipeline opportunities to find warm connections and identify gaps.

### Step 1 — Load all pipeline opportunities

```python
mcp__ronin__list_opportunities(user_id="<uuid>")
```

No state filter — networking is relevant at every pipeline stage.

### Step 2 — Receive contacts from user

The user provides contacts in one of two ways:

- **Inline:** "I know Sarah Chen at Stripe, she's a hiring manager. Email: sarah@stripe.com"
- **Batch:** "Here are my contacts: [list]" or "Use the contacts I copied from my contacts page"

Extract for each contact: `name`, `company`, `email` (if available), `relationship` type, and any notes.

### Step 3 — Match contacts to opportunities by company name

For each contact, compare `contact.company` against `opportunity.company` across all pipeline opportunities. Use case-insensitive matching and handle common variations:

- Exact match: "Stripe" == "Stripe"
- Subsidiary/parent: "Google" matches "Alphabet" (flag as probable match, not certain)
- Abbreviation: "AWS" matches "Amazon Web Services"

Build two lists:
- **Matched contacts** — contacts whose company maps to a pipeline opportunity
- **Unmatched contacts** — contacts at companies not in the pipeline (still valuable for future networking)

### Step 4 — Load company research for matched contacts

For each matched company, load research context:

```python
mcp__ronin__get_company_research(company_slug="stripe")
```

The `company_slug` is the company name lowercased with spaces replaced by hyphens. If research is not available (`found: false`), note the gap — the contact match is still valid, just without research context.

### Step 5 — Present cross-reference results

Structure the output as:

**Warm Connections (contacts matched to pipeline)**

For each match:
- "You know **[Name]** ([relationship]) at **[Company]** — you have a **[Title]** opportunity in **[state]** state."
- Include company research highlights if available (culture, tech stack, growth stage)
- Surface the contact's notes and how-we-met context if provided

**Gap Companies (pipeline opportunities with no contact match)**

For each unmatched opportunity:
- "[Company] — [Title] ([state]) — no contacts at this company"
- Prioritize by pipeline state: later states (interview_loop, offer) are more urgent gaps than early states (discovered)

**Unmatched Contacts (contacts not linked to pipeline)**

For each unmatched contact:
- "[Name] at [Company] — not in your pipeline"
- Note if the contact could be useful for future opportunities

### Step 6 — Return structured summary

Present findings in human-readable form. See [references/data-shapes.md](references/data-shapes.md) for the cross-reference result structure. Keep cross-reference results in the current conversation only. There is no backend contacts linkage or saved cross-reference state yet.

## Workflow: Generate Outreach

Create a personalized outreach message for a specific contact, grounded in the user's profile and company research.

### Step 1 — Load context (parallel)

Issue all context-loading calls in parallel:

```python
profile = mcp__ronin__get_profile(user_id="<uuid>")
memory = mcp__ronin__get_user_memory(user_id="<uuid>")
```

If the contact is matched to a pipeline opportunity, also load:

```python
research = mcp__ronin__get_research(user_id="<uuid>", opportunity_id="<opp-uuid>")
company = mcp__ronin__get_company_research(company_slug="stripe")
```

### Step 2 — Find shared background

Compare the user's profile against the contact's known details:

- **Same school** — check `profile.education` against contact's LinkedIn or known background
- **Overlapping companies** — check `profile.experiences[].company` against contact's company history
- **Shared skills/tech** — check `profile.skills` against the company's tech stack from research
- **Geographic overlap** — same city or region
- **Mutual connections** — from user memory or contact notes

If the profile is sparse, report the gap: "Limited shared background available — profile has [N] experiences and [N] skills. Consider enriching your profile for better personalization."

### Step 3 — Select message type

The user selects from five message types:

1. **Referral request** — asking the contact to refer them for a specific role
2. **Informational interview** — requesting a conversation to learn about the company/role
3. **Follow-up after application** — checking in after submitting an application
4. **Thank-you after interview** — post-interview gratitude with specific callbacks
5. **Re-engagement** — reconnecting with a contact not reached in 30+ days

Each type has a different tone, structure, and call-to-action.

### Step 4 — Generate personalized message

Extended thinking handles the synthesis. The model receives profile data, company research, relationship history from memory, and the selected message type.

Synthesis rules:

- **Lead with the shared background hook** — the strongest connection point goes in the opening line
- **Reference specific company context** — use culture signals, recent news, or tech stack from research
- **Match the relationship tone** — recruiter messages are more formal; colleague messages are casual
- **Include a clear ask** — every message ends with a specific, low-friction call-to-action
- **Keep it concise** — 3-5 short paragraphs maximum
- **Null over confabulation** — if no shared background exists, use a direct professional introduction instead of fabricating connections

Output format:

```text
Subject: [Generated subject line]

[Message body]
```

### Step 5 — Optionally send via Apple Mail

If the contact has an email address and Apple Mail MCP is available:

```python
mcp__apple_mail__compose_email(
    to="sarah@stripe.com",
    subject="Quick question about the Payments Platform team",
    body="Hi Sarah,\n\n..."
)
```

If Apple Mail MCP is not connected, present the message as copy-ready text.

### Step 6 — Check for prior correspondence (optional)

When Apple Mail MCP is available, search for existing email threads with the contact:

```python
mcp__apple_mail__search_emails(query="from:sarah@stripe.com")
```

Surface any prior correspondence to avoid redundant outreach or to reference previous conversations in the new message.

## Workflow: Network Gap Analysis

Identify pipeline companies where the user has no contacts and suggest strategies to close the gaps.

### Step 1 — Load pipeline and contacts

```python
mcp__ronin__list_opportunities(user_id="<uuid>")
```

Receive contacts from the user (same as Cross-Reference Workflow Step 2).

### Step 2 — Identify gap companies

Compare all pipeline opportunity companies against the contact list. A "gap" is a company in the pipeline with zero matching contacts.

### Step 3 — Prioritize gaps by pipeline state

Assign priority based on the pipeline state machine. Later states are more urgent:

| Priority | States | Rationale |
|----------|--------|-----------|
| Critical | `interview_loop`, `offer` | Active process — a warm connection could influence outcome |
| High | `applied`, `phone_screen` | Application submitted — referral could accelerate |
| Medium | `approved`, `materials_generated` | Preparing to apply — connection helps tailor materials |
| Low | `shortlisted` | Still evaluating — connection informs decision |
| Informational | `discovered` | Early stage — connection helps with research |

### Step 4 — Suggest networking strategies per gap

For each gap company, suggest actionable strategies:

- **LinkedIn search** — "Search for [Company] employees with [shared background] on LinkedIn"
- **Alumni network** — "Check [University] alumni directory for [Company] connections" (when profile has education data)
- **Conference/meetup** — "Look for [Company] engineers at upcoming [tech] meetups"
- **Second-degree connections** — "Ask [existing contact] if they know anyone at [Company]"
- **Cold outreach** — "Identify the hiring manager for [Title] on LinkedIn and send a personalized connection request"

### Step 5 — Present gap analysis

```text
NETWORK GAP ANALYSIS

Coverage: 3 of 10 pipeline companies have contacts (30%)

CRITICAL GAPS (active interviews)
  Datadog — Senior Engineer (interview_loop)
  -> Search LinkedIn for Datadog engineers from [your university]
  -> Ask [Contact Name] at [Related Company] for introductions

HIGH PRIORITY (applications submitted)
  Cloudflare — Staff Engineer (applied)
  -> Check alumni network for Cloudflare connections

MEDIUM PRIORITY (preparing to apply)
  Figma — Senior Frontend Engineer (approved)
  -> Look for Figma engineers at React Conf or similar events

LOW PRIORITY (evaluating)
  [remaining companies...]
```

## Error Handling

See [references/error-reference.md](references/error-reference.md) for the full error table, sparse data handling guidance, and roadmap. Key items:

- Empty opportunity list = run `ronin-discovery` first
- `get_company_research` returns `found: false` = cross-reference still works, just without research context
- Apple Mail connection refused = degrade gracefully to copy-ready text
- MCP connection unavailable = reconnect the Ronin MCP server configured for this harness, or fix the local API-mounted `/mcp` setup if you are explicitly running Ronin locally
- Sparse profile data = report limitation, generate outreach with available data only
