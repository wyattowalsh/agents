<Aside type="tip" title="What is MCP?">
The [Model Context Protocol](https://modelcontextprotocol.io/) is an open standard for connecting AI agents to external tools and data sources. Each server below exposes one or more tools that agents can invoke during a session.
</Aside>

<Aside type="note" title="Source of truth for this page">
This page is **hand-maintained** and currently summarizes the MCP server entries in the repository root `mcp.json`. That file includes a mix of local scripts, local packages, remote gateways, and third-party servers, so this list is broader than the repo-local `mcp/` directory.
</Aside>

## Managed client sync coverage

<Aside type="tip" title="Sync source of truth">
Edit `config/mcp-registry.json`, then preview local drift with `uv run python scripts/sync_agent_stack.py --targets home --check`. Apply repo and home updates with `uv run python scripts/sync_agent_stack.py --targets repo,home --apply`.
</Aside>

<Aside type="note" title="MCPHub local control plane">
MCPHub is now the preferred local MCP control plane. When `mcphub.enabled` is true in `config/mcp-registry.json`, clients receive MCPHub all, group, server, and disabled Smart Routing endpoints rooted at `http://127.0.0.1:46683`. Start it locally with `make mcphub-up`; Docker is not used for the managed launch path. ChatGPT remote MCP access uses the opt-in Cloudflare Tunnel URL from `.env.mcphub`.
</Aside>

<Aside type="note" title="Exact vs merge behavior">
`ChatGPT Desktop`, the GitHub Copilot shadow config, `Gemini antigravity`, the Gemini extension config, `OpenCode`, `AITK`, and `Crush` all receive the canonical baseline **and** preserve non-canonical extras already present in those files. `Claude Desktop`, `Cursor`, `Gemini CLI`, `GitHub Copilot CLI`, and `Codex` are rewritten to the exact canonical baseline. `Cherry Studio` receives a generated import pack in a tool-owned subdirectory instead of direct config mutation so unmanaged local MCP entries and ad hoc import bundles in the parent directory stay untouched.
</Aside>

<Steps>

1. Update the canonical MCP registry.

   Edit `config/mcp-registry.json` or the repo sources that feed it.

2. Preview repo and home drift.

   ```bash
   uv run python scripts/sync_agent_stack.py --targets repo,home --check
   ```

3. Apply the synchronized config set.

   ```bash
   uv run python scripts/sync_agent_stack.py --targets repo,home --apply
   ```

4. Import Cherry Studio MCP files only if you use Cherry Studio.

   Use `~/Library/Application Support/CherryStudio/mcp-import/managed/<server>.json` for additive one-server imports, or `~/Library/Application Support/CherryStudio/mcp-import/managed/all.json` only when you intentionally want to replace Cherry Studio's full managed MCP list.

   Top-level files already sitting in `~/Library/Application Support/CherryStudio/mcp-import/` are left alone intentionally.

</Steps>

## DuckDuckGo setup (representative client snippets)

Use server key `duckduckgo` with the `duckduckgo-mcp-server` package.

#### Standard JSON clients (`mcpServers`)

Applies to ChatGPT Desktop, Claude Desktop, Cursor, Gemini CLI, Gemini antigravity, and the Gemini extension config.

```json
{
  "mcpServers": {
    "duckduckgo": {
      "command": "uvx",
      "args": ["duckduckgo-mcp-server"]
    }
  }
}
```

#### Codex (`~/.codex/config.toml`)

```toml
[mcp_servers.duckduckgo]
command = "uvx"
args = ["duckduckgo-mcp-server"]
```

#### GitHub Copilot CLI (`~/.copilot/mcp-config.json`)

```json
{
  "mcpServers": {
    "duckduckgo": {
      "type": "stdio",
      "command": "uvx",
      "args": ["duckduckgo-mcp-server"],
      "tools": ["*"]
    }
  }
}
```

#### OpenCode (`~/.config/opencode/opencode.json`)

```json
{
  "mcp": {
    "duckduckgo": {
      "type": "local",
      "command": ["uvx", "duckduckgo-mcp-server"],
      "enabled": true
    }
  }
}
```

#### AITK (`~/.aitk/mcp.json`)

```json
{
  "servers": {
    "duckduckgo": {
      "type": "stdio",
      "command": "uvx",
      "args": ["duckduckgo-mcp-server"]
    }
  }
}
```

## Chrome DevTools setup (representative client snippets)

Use server key `chrome-devtools` through the managed MCPHub wrapper. The wrapper starts or reuses a dedicated visible Chrome on `127.0.0.1:9333` with profile `~/.cache/chrome-devtools-mcp-login`, then attaches `chrome-devtools-mcp` with `--browserUrl`. This avoids the automation launch flags that `chrome-devtools-mcp` passes when it launches Chrome itself. Chrome opens lazily when the client invokes a browser-backed tool such as `list_pages`, `navigate_page`, or `take_screenshot`; restart the MCP client after config changes so it does not keep using an already-started server process.

Replace `<repo>` in the examples below with the absolute path to your local checkout.

#### Standard JSON clients (`mcpServers`)

Applies to ChatGPT Desktop, Claude Desktop, Cursor, Gemini CLI, Gemini antigravity, and the Gemini extension config.

```json
{
  "mcpServers": {
    "chrome-devtools": {
      "command": "bash",
      "args": ["<repo>/scripts/mcphub/chrome-devtools-browser-url.sh"]
    }
  }
}
```

#### Codex (`~/.codex/config.toml`)

```toml
[mcp_servers.chrome-devtools]
command = "bash"
args = ["<repo>/scripts/mcphub/chrome-devtools-browser-url.sh"]
```

#### GitHub Copilot CLI (`~/.copilot/mcp-config.json`)

```json
{
  "mcpServers": {
    "chrome-devtools": {
      "type": "stdio",
      "command": "bash",
      "args": ["<repo>/scripts/mcphub/chrome-devtools-browser-url.sh"],
      "tools": ["*"]
    }
  }
}
```

#### OpenCode (`~/.config/opencode/opencode.json`)

```json
{
  "mcp": {
    "chrome-devtools": {
      "type": "local",
      "command": ["bash", "<repo>/scripts/mcphub/chrome-devtools-browser-url.sh"],
      "enabled": true
    }
  }
}
```

#### Crush (`~/.config/crush/crush.json`)

```json
{
  "mcp": {
    "chrome-devtools": {
      "type": "stdio",
      "command": "bash",
      "args": ["<repo>/scripts/mcphub/chrome-devtools-browser-url.sh"]
    }
  }
}
```
